# Evidencias de calidad

Agrega aqui las capturas y archivos generados durante la entrega final.

## Capturas sugeridas

1. Aplicacion funcionando localmente.
2. Aplicacion desplegada en Vercel.
3. Resultado de `npm test`.
4. Resultado de `npm run build`.
5. Ejecucion de GitHub Actions.
6. Pruebas Postman contra la URL de Vercel.
7. Prueba JMeter contra la URL de Vercel.
8. Consola del navegador sin errores visibles.

## Checklist de aceptacion

| ID | Criterio | Cumple | Observaciones |
| --- | --- | --- | --- |
| 1 | El sitio educativo se despliega correctamente en Vercel sin errores visibles. | Pendiente | Completar despues del despliegue. |
| 2 | El proyecto carga en menos de 3 segundos desde Vercel. | Pendiente | Medir con navegador o JMeter. |
| 3 | El endpoint `/api/lessons.json` devuelve datos correctos. | Pendiente | Validar con Postman. |
| 4 | La navegacion e interaccion del aplicativo es fluida. | Pendiente | Validar manualmente. |
| 5 | No se presentan errores visibles en consola del navegador. | Pendiente | Adjuntar captura. |
| 6 | Las pruebas unitarias pasan correctamente en el pipeline automatico. | Pendiente | Adjuntar captura de GitHub Actions. |
| 7 | Las pruebas de integracion con Postman son exitosas. | Pendiente | Adjuntar resultado exportado. |
| 8 | La prueba JMeter tiene promedio menor a 5 segundos. | Pendiente | Adjuntar reporte. |
